launch - Run MPI commands¶

Usage¶

e4s-cl launch [ OPTIONS ] [ MPI LAUNCHER [ -- ] ] < COMMAND >

Options¶

The launch command accepts several options to tune the outcome of its execution:

--profile

Profile to use for the execution

--image

Path for the image to use when launching the container

--backend

Container technology to employ

--libraries

Comma-separated list of libraries to load at run-time

--files

Comma-separated list of files to make accessible in the container

--source

Path of script to source before execution

MPI launcher detection

E4S Container Launcher needs to determine the exact arguments of the launcher. Certain launchers may have unique options that e4s-cl may not recognize.

To ensure the arguments are understood correctly, an additional -- can be added between the launcher options and the command.

See the examples for an illustration.

Description¶

E4S Container Launcher is a accessory launcher to ensure host MPI libraries are used in containers. It wraps around a valid MPI launch command to work.

The minimal options that must be given in order to run are:

  • A container image;

  • A container technology to run the image with.

Other options then influence the execution:

  • Arguments passed to --files will be made available in the container;

  • Libraries passed to --libraries will be loaded;

  • A script passed to --source will be run in the container before any other command.

All of these options can be bypassed by passing a profile. The fields of the target profile are then implicitly used for each of the above options.

Using a selected profile

When a profile is selected, it will be used if no --profile option is passed. Command line options have precedence over profiles’ fields.

Implicit sub-command

When e4s-cl is called without a proper sub-command (launch or profile), the program will implicitly use the launch sub-command. This requires a complete profile to be selected as no launch arguments can be passed.

Examples¶

A launch command using an explicit profile and an MPI launcher:

e4s-cl launch --profile intel-21 mpirun -np 2 ./ping-pong

A launch command using an explicit profile and an MPI launcher, but a different image than the profile’s:

e4s-cl launch --profile intel-21 \
    --image /home/user/ecp.simg \
        mpirun -np 2 ./ping-pong

A launch command with implicit profile making explicit the launcher and launchee:

e4s-cl launch mpirun -np 2 -ppn 1 -- ./ping-pong

A launch command importing binaries in the container before running a script without profile:

e4s-cl launch --files /bin/strace,/bin/lsof \
    --backend singularity \
    --image ~/Images/ecp.simg \
        ./script.sh

A launch command using a local mpich library without profile:

e4s-cl profile launch --backend singularity \
     --image /home/user/ecp.simg \
     --libraries /home/user/spack/opt/spack/linux-arch-skylake_avx512/gcc-10.2.0/mpich-3.4.1-yjx3whq2g2mrzrws4xhoxyjt7hl6wvb5/lib/libmpi.so.12 \
     --files /usr/share/hwdata/pci.ids \
         mpirun -np 2 ./ping-pong

A implicit launch command (parameters implicitly passed via selected profile):

e4s-cl mpirun -np 2 ./ping-pong
